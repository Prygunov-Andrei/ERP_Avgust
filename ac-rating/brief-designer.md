**Инструмент:** Claude Design (claude.ai/design)
**Дата:** 2026-04-18
**Авторы brief:** Андрей Прягунов + Claude (tech lead)
**Исполнители дизайна:** Андрей + Максим совместно в Claude Design

> Этот документ — первичный brief для старта проекта в Claude Design. Ты (Claude Design) будешь вести проект через чат + inline-комментарии на canvas. В конце — handoff готовых экранов в Claude Code для реализации.

## 1. Goal (зачем)

Создать **единый визуальный язык публичной части сайта** компании «Август Климат» — производителя и подбора климатической техники (вентиляция, кондиционирование).

Публичная часть — **единый сайт `hvac-info.com`**, который со временем вырастет до **7-12 разделов**. На старте активны 3 (остальные планово):

**Сейчас в header:**
- **Новости HVAC-индустрии** (`/news`, текущий landing) — блок о производителях, выставках, трендах. Уже живёт на продакшне.
- **Рейтинг кондиционеров «Август-климат»** (`/ratings`) — независимый рейтинг бытовых сплит-систем. Наш целевой проект.
- **Оценка сметы** (`/estimate`) — отдельный продукт для расчёта стоимости монтажа. За пределами этого brief-а, но **место в навигации резервируем**.

**Будущие разделы** (подтверждённые владельцем, без конкретных сроков запуска):

| Раздел | URL (предв.) | Тип контента | На что похоже |
|--------|--------------|--------------|---------------|
| **Биржа монтажников** | `/installers` | Маркетплейс профилей исполнителей: фильтры по региону/специализации/рейтингу, карточка профиля, форма заявки на услугу | Job-board / Avito-профили |
| **Анализ проектов** | `/analytics` | Портфолио кейсов / калькулятор расчётов / аналитические инструменты | Кейсы-стори + интерактив |
| **Франшиза Август-климат** | `/franchise` | Продающий лендинг для привлечения партнёров-франчайзи, форма заявки, преимущества, условия | B2B-лендинг |
| **Ассоциация климатехников России** | `/association` | Информационно-общественный раздел: участники, новости ассоциации, правила, членство, подписка | Info-портал |

Плюс ещё 3-5 позиций могут появиться позже.

Дизайн-система **ОБЯЗАНА** выдержать рост до 10-12 разделов без переделки. **И выдержать четыре принципиально разных типа контента** — маркетплейс, портфолио, лендинг, информационный портал — помимо текущих новостей и рейтинга.

⚠ **КРИТИЧНОЕ ТРЕБОВАНИЕ — МАСШТАБИРУЕМОСТЬ.** Это не trim-сайт на 3 страницы. Дизайн нужно строить как **масштабируемую платформу**:
1. **Навигация** не может быть простой горизонтальной строкой — она развалится на 7+ пунктах. Предложи паттерн: mega-menu с группами, или двухуровневая (логотип + основное меню + верхняя утилитарная строка), или burger/drawer даже на desktop. На мобиле — однозначно drawer.
2. **Универсальные шаблоны страниц** — листинг (карточки + фильтры), деталь (сайдбар + контент), landing раздела, форма, статика. Любой новый раздел должен складываться из этих шаблонов за вечер.
3. **Компоненты контента** — карточка товара / новости / кондиционера / исполнителя должны быть одним семейством компонентов с общей сеткой и пропорциями.
4. **Цветовая и типографическая система** — токены, а не конкретные цвета в компонентах. Новый раздел не должен требовать новых цветов.

Цель — все разделы должны выглядеть как **один продукт**, а не как склейка. Сейчас есть только новости и они выглядят как «dev-дизайн без дизайнера» — ты (Claude Design) делаешь дизайн-систему, под которую подтянутся и новости (редизайн), и новый рейтинг, и все следующие 5-10 разделов.

## 2. Audience (для кого)

- **B2C-покупатели климатической техники.** Средний чек по рейтингу — 19 000–155 000 ₽. Это не импульсные покупки, аудитория сравнивает, читает отзывы, думает неделями.
- Типичный пользователь: владелец квартиры / загородного дома, выбирает кондиционер «на лето».
- Вторичная аудитория — профессиональные монтажники и HVAC-инженеры (им нужны лабораторные данные, они ценят детализацию).

**Поэтому дизайн должен транслировать:** независимость, данные, доверие, без «маркетинговой показухи».

## 3. Scope (что нарисовать)

### Рейтинг кондиционеров (основной блок)

| # | Страница | Роут | Описание |
|---|----------|------|----------|
| 1 | **Главная рейтинга** | `/ratings` | Hero-секция + фильтры + листинг моделей (таблица/карточки/гибрид — твой дизайн-выбор) + 3 таба «По индексу / Самые тихие / Свой рейтинг» |
| 2 | **Деталь модели** | `/ratings/[slug]` | Галерея фото + параметры с визуализацией вклада в индекс + видео + плюсы/минусы + «где купить» + отзывы + форма отзыва |
| 3 | **Методика** | `/ratings/methodology` | Описание критериев оценки (33 критерия с весами), как считается индекс |
| 4 | **Форма заявки** | `/ratings/submit` | Длинная форма «Добавить мой кондиционер в рейтинг» — 20+ полей, разбитых по разделам |
| 5 | **Тихие** | `/ratings/quiet` | Отдельный листинг, сортировка по шуму (almost та же страница что главная, но с акцентом на шум) |
| 6 | **Архив** | `/ratings/archive` | Модели, снятые с рейтинга. Сейчас empty — empty-state важен |

### HVAC-новости (редизайним в том же стиле)

| # | Страница | Роут | Описание |
|---|----------|------|----------|
| 7 | **Листинг новостей** | `/news` или `/` | Карточки новостей, пагинация, фильтры (по производителю, рейтингу AI, месяцу) |
| 8 | **Деталь новости** | `/news/[id]` | Статья с медиа, связанные новости |

### Общие элементы (дизайн-система уровня платформы)

- **Header / навигация** — **масштабируемая**. Должна выдержать 7-12 пунктов верхнего уровня + подуровни. Нарисуй варианты:
  - Desktop «сейчас» (3-5 пунктов)
  - Desktop «через год» (8-10 пунктов) — проверка расширяемости
  - Mobile drawer с вложенностью
- **Footer** — полноценный, с колонками по разделам. Тоже должен дышать при росте числа разделов.
- **Универсальные шаблоны страниц** (это отдельный deliverable — сразу выдели в style guide):
  - Шаблон «листинг» (карточки/таблица + фильтры + пагинация + sort + view-toggle)
  - Шаблон «деталь» (breadcrumbs + заголовок + content-area + sidebar/aside + related)
  - Шаблон «landing раздела» (hero + секции + CTA)
  - Шаблон «форма» (многоуровневая, с секциями и прогрессом)
  - Шаблон «статика» (методика, about, политики)
- **Семейство карточек** — одна визуальная система для: новости / кондиционера / модели в архиве / производителя / поставщика / исполнителя / услуги / etc. Меняется только content, структура — общая.
- **Empty states** — унифицированный компонент (иконка + заголовок + подзаголовок + CTA)
- **Loading skeletons** для SSR hydration
- **404 / 500** — фирменные страницы ошибок
- **Toast / notifications** — для отправки отзыва / заявки

## 4. Tech stack (критично — учти при дизайне)

Цель реализации — Next.js 16 App Router поверх существующего фронта ERP.

**Готовые компоненты** (опирайся на них, не изобретай):
- **Tailwind CSS 4** через `@theme` директивы. Токены — HSL CSS variables (`--primary`, `--accent`, `--destructive`, `--muted`, `--background`, `--card`, `--border`, `--ring`).
- **shadcn/ui** — 22 примитива уже в проекте: Accordion, Alert, Badge, Button, Calendar, Card, Checkbox, Dialog, Dropdown-menu, Input, Label, Progress, Radio-group, Select, Separator, Skeleton, Slider, Switch, Table, Tabs, Textarea, Tooltip.
- **Иконки** — Lucide React. Не используй другие библиотеки.
- **Тема** — light / dark через class-based switching (ThemeProvider уже в коде).

**Не надо рисовать:** свои кастомные кнопки/инпуты в другом стиле. Если нужен компонент, которого нет в shadcn/ui (donut-chart, rating-stars, gauge), — явно отметь в спеке.

Ссылки на код-базу для Claude Design:

- **Репозиторий ERP:** `https://github.com/Prygunov-Andrei/ERP_Avgust`, ветка `main`, путь `frontend/`

## 6. UX-структура, которую НЕ ломать

Максим продумал воронку и логику. Сохраняем:

1. **Три таба на главной рейтинга:** «По индексу Август-климат», «Самые тихие», «Пользовательский рейтинг» (пользователь отключает критерии — индекс пересчитывается).
2. **Визуализация вклада в индекс** на карточке критерия: название + значение + единица + вес × балл = вклад + прогресс-бар.
3. **Пометка «выше эталона»** на параметрах, где модель бьёт рекорд.
4. **Плюсы / Минусы** отдельным блоком на деталке.
5. **Галерея фото** с thumbnails и lightbox (клавиатурная навигация).
6. **Видеообзоры** — embed YouTube / VK.
7. **Блок «Где купить»** — ссылки на поставщиков.
8. **Форма отзыва** со звёздным рейтингом 1-5, плюсы / минусы / комментарий, премодерация.
9. **Форма подачи модели** разбита на секции (модель / характеристики / теплообменники / подтверждение замеров / контакт / согласие на ПДн).

## 7. Требования к дизайну

### Темы
- **Dark mode — обязательно.** Оба варианта (light/dark) для всех экранов.
- Переключатель темы в header.

### Адаптив
- **Desktop 1440×900** — эталон.
- **Mobile 390×844** — минимум.
- Tablet (768) — желательно, если не успеваем — пусть будет автоматически между desktop и mobile.
- **Особое внимание — мобильный листинг рейтинга.** Сейчас таблица ломается (см. `v2-04-home-mobile.png`): колонки «Бренд» и «Индекс» становятся нечитаемыми. Предложи принципиально другое решение для мобилки (карточки / accordion / горизонтальный скролл — выбирай).

### Состояния
Нарисовать для каждого интерактивного компонента:
- `default`, `hover`, `focus`, `active`, `disabled`
- Для списков: `loading` (skeleton), `empty`, `error`

### Визуальные акценты, которых не хватает сейчас

| Что сейчас | Что должно быть |
|------------|-----------------|
| Hero нет — сразу таблица | Hero с объяснением «Что такое индекс Август-климат» + цифры (N моделей, M критериев, K лет замеров) + CTA |
| Индекс как `78.8 / 100` синим текстом | Визуальная шкала: donut / gauge / stacked bar — что-то запоминающееся |
| Разнокалиберные лого брендов в таблице | Нормализованный контейнер для лого (единая рамка / фон / размер) |
| Прогресс-бары на деталке все зелёные | Градация зелёный / жёлтый / красный в зависимости от относительного положения модели среди всех |
| Пустой архив = «Архивных моделей пока нет» | Продуманный empty-state с объяснением «Рейтинг обновляется раз в полгода, архив появится после первой ротации» |
| Footer из 3 ссылок | Информативный footer с контактами, методикой, юр. информацией, соцсетями Август-климат |
| Никакого сравнения моделей | Возможность выбрать 2-3 модели чекбоксами и открыть страницу сравнения side-by-side |

### Брендинг

- **Название:** Август-климат / Август Климат (оба написания встречаются — определи одно).
- **Логотип:** есть в `frontend/public/logo.png` в репо ERP. Если кажется слабым — сделай альтернативу (но это отдельный продакт-вопрос).
- **Цветовая палитра:** сейчас в ERP используется нейтральная shadcn-база. Для публичной части можно вывести акцентный цвет (синий — ассоциация с прохладой, белый — чистота), но не противореча ERP-токенам.

## 8. Формат выдачи (handoff)

**Основной путь:** Claude Design → Claude Code (web или local agent).

Когда страницы готовы:
1. Экспортируем каждый ключевой экран (страница + темы + адаптив) в отдельный frame.
2. Handoff в Claude Code → генерится React / Tailwind / shadcn код.
