# Каталоги поставщиков — Руководство пользователя

## Назначение

Модуль «Каталоги поставщиков» позволяет загружать PDF-каталоги от производителей инженерного оборудования и автоматически извлекать из них номенклатуру товаров с размерными вариантами. Извлечённые товары сохраняются в JSON и могут быть импортированы в Номенклатуру ERP.

**Поддерживаемые типы каталогов:**
- Каталоги вентиляционного оборудования (воздуховоды, вентиляторы, нагреватели, фильтры и т.д.)
- Каталоги кабеленесущих систем (лотки, короба, аксессуары)
- Каталоги комплектующих (крепёж, теплоизоляция, огнезащита)
- Каталоги решёток и диффузоров
- Любые другие каталоги инженерного оборудования с табличными данными

---

## Как открыть

В боковом меню перейдите: **Товары и услуги** → **Каталог закупок** → **Каталоги поставщиков**

---

## Пошаговая инструкция

### Шаг 1. Загрузка PDF-каталога

1. На странице «Каталоги поставщиков» нажмите кнопку **«Загрузить каталог»**
2. В открывшемся диалоге заполните:
   - **Название каталога** — понятное название, например «Каталог WHEIL Канальная вентиляция v23.1»
   - **Код поставщика** — короткий идентификатор латиницей, например `wheil`, `galvent`, `breez`
   - **PDF-файл** — перетащите файл в зону загрузки или нажмите для выбора
3. Нажмите **«Загрузить»**

После загрузки каталог появится в таблице со статусом **«Загружен»** и указанием количества страниц.

### Шаг 2. Определение оглавления

1. Откройте каталог (нажмите на название в таблице)
2. Нажмите кнопку **«Определить оглавление»**
3. Система отправит первые 6 страниц PDF в нейросеть, которая:
   - Определит разделы каталога (по типам оборудования)
   - Укажет диапазон страниц для каждого раздела
   - Сопоставит разделы с существующими категориями товаров в системе
   - Если подходящей категории нет — предложит создать новую

**Время выполнения:** 10–30 секунд.

После завершения вы увидите таблицу секций:

| Раздел | Страницы | Категория | Статус |
|--------|----------|-----------|--------|
| Канальные вентиляторы | 10–25 | `vent_fans` | Существующая |
| Листовые кабельные лотки | 5–18 | `electrical_trays_sheet` | **Будет создана** |

- **Зелёный бейдж** — категория уже существует в системе
- **Жёлтый бейдж** — категория будет создана автоматически при парсинге

### Шаг 3. Проверка и редактирование секций

Перед парсингом рекомендуется проверить предложенные секции:

- **Название раздела** — можно отредактировать
- **Диапазон страниц** — можно скорректировать, если нейросеть ошиблась
- **Код категории** — можно изменить на другую существующую категорию

После редактирования нажмите **«Сохранить секции»**.

### Шаг 4. Запуск парсинга

1. Нажмите кнопку **«Запустить парсинг»**
2. Процесс запускается в фоновом режиме — вы можете продолжать работу
3. Прогресс отображается на странице каталога:
   - Прогресс-бар с процентом выполнения
   - Текущая секция и батч
   - Количество найденных товаров и вариантов

**Время выполнения:** зависит от размера каталога:
- 40 страниц — ~5 минут
- 60 страниц — ~10 минут
- 120 страниц — ~20–30 минут

Во время парсинга можно нажать **«Отменить»**, чтобы остановить процесс. Уже обработанные данные сохранятся.

### Шаг 5. Проверка результатов

После завершения парсинга (статус **«Разобран»**):
- Проверьте общее количество товаров и вариантов
- Скачайте JSON-файл для просмотра (кнопка **«Скачать JSON»**)
- Если есть ошибки — они отображаются в секции «Ошибки» с указанием проблемных страниц

### Шаг 6. Импорт в Номенклатуру

1. Нажмите кнопку **«Импортировать в БД»**
2. Если товары этого поставщика уже были импортированы ранее — установите флажок **«Заменить существующие»** (старые товары поставщика будут удалены)
3. После подтверждения:
   - Автоматически создаются недостающие категории
   - Каждый размерный вариант становится отдельным товаром в Номенклатуре
   - Для каждого товара создаётся алиас с базовым именем (для будущего матчинга по счетам)
   - Добавляется маркер поставщика для возможности повторного импорта

После успешного импорта (статус **«Импортирован»**) товары появятся в разделе **Товары и услуги** → **Номенклатура**.

## Скачивание файлов

На странице каталога доступны ссылки для скачивания:
- **PDF** — исходный файл каталога
- **JSON** — результат парсинга (после завершения парсинга)

В таблице каталогов также есть иконки скачивания для каждого каталога.

## Статусы каталога

| Статус | Описание |
|--------|----------|
| **Загружен** | PDF загружен, ожидает обработки |
| **Анализ оглавления** | Идёт определение секций через нейросеть |
| **Оглавление готово** | Секции определены, можно запускать парсинг |
| **Парсинг** | Идёт извлечение товаров из PDF |
| **Разобран** | Парсинг завершён, JSON готов к импорту |
| **Импорт в БД** | Идёт запись товаров в Номенклатуру |
| **Импортирован** | Товары успешно импортированы |
| **Ошибка** | Произошла ошибка (см. описание) |

## Что делать при ошибках

### «Не удалось определить оглавление»
- Попробуйте увеличить количество страниц для анализа (по умолчанию 6)
- Убедитесь, что PDF содержит оглавление или содержание на первых страницах
- Если каталог без оглавления — попробуйте указать секции вручную

### «Ошибка парсинга на страницах X–Y»
- Страницы могут содержать только изображения без текста/таблиц
- Перезапустите парсинг — обычно повторный запуск успешен
- Если ошибка повторяется — отредактируйте секции, исключив проблемные страницы

### «Категория не найдена»
- Проверьте правильность кода категории в секциях
- При необходимости создайте категорию вручную через **Товары и услуги** → **Категории**

### Парсинг идёт слишком долго
- Крупные каталоги (100+ страниц) могут обрабатываться до 30 минут
- Если процесс завис — отмените и перезапустите

## Часто задаваемые вопросы

**Можно ли загрузить один каталог повторно?**
Да. Загрузите PDF заново — система создаст новую запись. При импорте в БД используйте опцию «Заменить существующие», чтобы обновить товары.

**Какие форматы PDF поддерживаются?**
Любые PDF-файлы. Лучшие результаты — каталоги с текстовыми таблицами. Каталоги со сканированными страницами также поддерживаются через OCR нейросети.

**Как добавить нового поставщика?**
Укажите новый код поставщика (латиницей) при загрузке каталога. Код используется для группировки товаров и управления импортом.

**Можно ли редактировать JSON после парсинга?**
Скачайте JSON, отредактируйте вручную и загрузите каталог заново. Прямое редактирование JSON в системе пока не поддерживается.
