# Инструкции: Справочники и настройки

## Персонал

Раздел «Персонал» доступен через главное меню → Справочники и настройки → Персонал.

### Список сотрудников

На главной странице отображается таблица со всеми сотрудниками компании. Доступны фильтры по статусу (активные/уволенные), а также строка поиска по ФИО. Нажмите на строку сотрудника, чтобы открыть его карточку для редактирования.

### Создание сотрудника

1. Нажмите кнопку **«+ Добавить сотрудника»**.
2. Заполните обязательное поле **ФИО**.
3. Укажите должность, дату рождения, дату приёма, пол.
4. При необходимости привяжите **учётную запись** — это позволит сотруднику входить в систему.
5. Нажмите **«Сохранить»**.

### Привязка учётной записи (User)

Поле «Учётная запись (User)» позволяет связать запись сотрудника с учётной записью в системе. Это обязательно для того, чтобы сотрудник мог войти в ERP и его права доступа определялись автоматически.

- Выпадающий список показывает только те учётные записи, которые ещё не привязаны к другим сотрудникам.
- Чтобы отвязать учётную запись, выберите «Не привязана».
- У одного сотрудника может быть только одна учётная запись, и наоборот.

### Вкладки карточки сотрудника

- **Основное** — ФИО, контактные данные, привязка учётной записи, должность, статус.
- **Должности** — история назначений в различных юридических лицах. Текущая должность помечена зелёным.
- **Зарплата** — история изменений оклада (полный и официальный).
- **Банк** — реквизиты для перечисления зарплаты (банк, БИК, корр. счёт, р/с, номер карты).
- **Связи** — руководители и подчинённые (иерархия).
- **Права** — настройка прав доступа к разделам ERP.

### Оргструктура

Вкладка «Оргструктура» (доступна из списка сотрудников) визуально показывает иерархию подчинения в виде дерева.

## Права доступа ERP

Каждому сотруднику назначаются права на разделы и подразделы ERP-системы на вкладке «Права» в карточке сотрудника.

### Уровни доступа

| Уровень | Описание |
|---------|----------|
| **Нет доступа** | Раздел/подраздел полностью скрыт из меню, переход по URL невозможен |
| **Чтение** | Раздел виден, данные доступны для просмотра, но без возможности редактирования |
| **Редактирование** | Полный доступ — просмотр, создание, изменение, удаление |

### Разделы и подразделы

Права назначаются как на уровне **раздела целиком**, так и на уровне **каждого подраздела** отдельно:

| Раздел | Подразделы |
|--------|------------|
| Пункт управления | — |
| Коммерческие предложения | Канбан КП, ТКП, МП, Сметы, Прайс-листы |
| Объекты | — |
| Финансы | Дашборд, Платежи, Выписки, Периодические платежи, Дебиторская задолженность, Бухгалтерия, Расходный бюджет, Показатели |
| Договоры | Рамочные договоры, Договоры по объектам, Сметы, Монтажные сметы, Акты, Хозяйственные договоры |
| Снабжение и Склад | Канбан снабжения, Счета на оплату, Календарь водителей, Модерация товаров, Склад |
| ПТО | Проекты, Производственная документация, Исполнительная документация, Образцы, Руководящие документы |
| Маркетинг | Канбан поиска объектов, Потенциальные заказчики, Поиск исполнителей |
| Переписка | — |
| Справочники и настройки | Товары и услуги, Фронт работ, Персонал, Контрагенты, Настройки |
| Справка | — |

### Каскадное назначение

При изменении уровня доступа к **разделу** все его подразделы автоматически обновляются на тот же уровень. После этого можно точечно изменить права на отдельные подразделы.

Например: установить «Справочники и настройки» → Чтение, а затем «Персонал» → Нет доступа. Сотрудник будет видеть все справочники, кроме раздела «Персонал».

### Специальные разрешения

- **Одобрение платежей** — возможность утвердить платёж в реестре оплат.
- **Одобрение счетов** — возможность утвердить счёт от поставщика.
- **Администрирование канбана** — настройка колонок и этапов.

### Как назначить права

1. Откройте карточку сотрудника.
2. Перейдите на вкладку **«Права»**.
3. Для каждого раздела выберите нужный уровень доступа — все подразделы обновятся каскадно.
4. При необходимости точечно измените права на отдельные подразделы.
5. Нажмите **«Сохранить»**.

> **Важно:** Администратор (superuser) имеет полный доступ ко всем разделам вне зависимости от настроек прав.

## Мои компании (Юридические лица)

Раздел доступен через Справочники и настройки → Настройки → вкладка «Мои компании».

### Создание юрлица

1. Нажмите **«+ Добавить компанию»**.
2. Заполните: название, ИНН (обязательно), КПП, ОГРН.
3. Выберите систему налогообложения.
4. При необходимости укажите данные директора для генерации документов.

### Редактирование / Удаление

Используйте контекстное меню (⋮) на карточке юрлица.

---

## Банковские счета

Раздел доступен через Справочники и настройки → Настройки → вкладка «Счета».

### Создание счёта

1. Нажмите **«+ Добавить счёт»**.
2. Выберите юридическое лицо.
3. Укажите тип (расчётный, касса, депозит, валютный).
4. Заполните номер счёта, банк и БИК.
5. Если требуется — укажите начальный остаток.

### Получение остатка из банка

Для привязанных счетов доступна функция автоматического получения остатка через API банка. Используйте кнопку «Получить остаток из банка» в контекстном меню.

## Категории расходов

Раздел доступен через Справочники и настройки → Настройки → вкладка «Категории расходов».

Категории используются при создании платежей для классификации расходов. Поддерживается иерархическая структура (родительская-дочерняя).

### Создание категории

1. Нажмите **«+ Добавить категорию»**.
2. Укажите название и код.
3. При необходимости выберите родительскую категорию.
4. Включите флаг «Требует привязки к договору» если при оплате обязательна привязка.

---

## Налоговые системы

Раздел доступен через Справочники и настройки → Настройки → вкладка «Налоговые системы».

Используется при создании юридических лиц для указания системы налогообложения (ОСН, УСН 6%, УСН 15% и т.д.).

---

## LLM-провайдеры

Раздел доступен через Справочники и настройки → Настройки → вкладка «LLM-провайдеры».

Настройка API-ключей для языковых моделей, используемых системой для анализа документов (импорт смет, разбор счетов и т.д.).

---

## Банковские подключения

Раздел доступен через Справочники и настройки → Настройки → вкладка «Банковские подключения».

Настройка подключений к банковским API для автоматического получения выписок и остатков.
