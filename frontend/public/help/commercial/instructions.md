# Инструкции: Коммерческие предложения

## Общий бизнес-процесс

Бизнес-процесс коммерческих предложений охватывает путь от первого контакта с потенциальным клиентом до подготовки полного пакета документов КП. Процесс проходит через два связанных канбана:

1. **Канбан поиска объектов** (раздел «Маркетинг») — начальный этап: поиск клиентов, назначение встреч, передача объекта на расчёт.
2. **Канбан КП** (раздел «Коммерческие предложения») — расчётный этап: подготовка смет, запрос счетов у поставщиков, утверждение и формирование КП.

### Механизм «Тоннель»

Оба канбана работают на одной доске `commercial_pipeline` с 12 колонками. Каждый канбан показывает только свои колонки:

| Маркетинг (6 колонок) | КП (6 колонок) |
|---|---|
| Новые клиенты | Новый расчёт |
| Назначена встреча | В работе |
| Проведена встреча | Счета запрошены |
| Расчёт подготовлен | Утверждение сметы |
| Нет результата | Смета утверждена |
| Есть результат | Подготовлено КП |

Когда карточка перемещается из **«Проведена встреча»** → **«Новый расчёт»**, она «исчезает» из канбана Маркетинга и появляется в канбане КП (первый столбец).

Когда расчёт готов и карточка перемещается в **«Подготовлено КП»** → **«Расчёт подготовлен»**, она возвращается обратно в канбан Маркетинга со всеми накопленными данными.

## Работа с канбаном КП

### Карточка канбана

Каждая карточка содержит:

- **Номер КП** — заголовок
- **Наименование объекта** — из справочника объектов
- **Система** — текстовое описание (например, «Вентиляция», «Кондиционирование»)
- **Потенциальный заказчик** — отображается третьей строкой на карточке канбана и в детальном виде
- **Счета поставщиков** — файлы (PDF, Excel), полученные от поставщиков в процессе расчёта
- **Комментарии** — оперативные заметки
- **Привязанные ТКП** — ссылки на технико-коммерческие предложения

### Перемещение карточек

Используйте выпадающий список внизу каждой карточки для перемещения между колонками. Карточку можно перемещать в любую колонку всей доски (не только видимых).

## Список ТКП

Таблица списка ТКП (`/proposals/technical-proposals`) содержит колонки: Номер, Название, Создано, Дата выдачи, Объект, Компания, Статус, Сумма, Версия.

- **Клик по строке** открывает детальную страницу ТКП
- ТКП с подходящим сроком выдачи (**<=3 дней**) подсвечиваются **оранжевым**, просроченные — **красным**
- Фильтрация по объекту, компании, статусу и поиск по номеру/названию

## Создание ТКП (Технико-Коммерческое Предложение)

### Шаг 1: Основные поля

1. Откройте раздел **Коммерческие предложения → ТКП**
2. Нажмите **«Создать ТКП»**
3. Заполните обязательные поля:
   - **Название** — краткое описание ТКП
   - **Дата** — дата создания
   - **Дата выдачи (крайний срок)** — срок, к которому ТКП должно быть выдано Заказчику. Необязательное поле. Если указано, ТКП с подходящим сроком (<=3 дней) подсвечиваются оранжевым в списке, а просроченные — красным.
   - **Объект** — выберите из справочника
   - **Компания** — ваше юрлицо

### Шаг 2: Привязка смет

1. Перейдите на вкладку **«Сметы»**
2. После выбора объекта отобразятся все сметы, созданные по этому объекту
3. Отметьте галочками нужные сметы (можно выбрать несколько)
4. Данные из смет будут автоматически скопированы в ТКП при сохранении

### Шаг 3: Фронт работ

1. Перейдите на вкладку **«Фронт работ»**
2. Пункты, отмеченные как «по умолчанию» в справочнике, уже выбраны
3. Вы можете:
   - Снять галочку с ненужных пунктов
   - Добавить дополнительные пункты
   - Указать сроки (текстом или конкретной датой)

### Шаг 4: Сохранение

Нажмите **«Создать»**. Система автоматически:
- Сгенерирует номер ТКП
- Привяжет выбранные сметы
- Создаст записи фронта работ

## Детальная страница ТКП

На детальной странице ТКП доступны следующие вкладки:

- **Основная информация** — общие данные, финансовая информация (продажа, закупка, прибыль, трудозатраты)
- **Сметы** — привязанные сметы объекта, сгруппированные по смете с детализацией разделов
- **Разделы** — разворачиваемые разделы с подразделами. Каждый подраздел показывает: материалы продажа/закупка, работы продажа/закупка, итого, прибыль. Суммы подразделов можно редактировать, нажав на иконку карандаша
- **Характеристики** — добавление, редактирование и удаление характеристик с расчётом прибыли
- **Фронт работ** — элементы фронта работ из справочника

Навигационные хлебные крошки показывают номер ТКП (например «ТКП 221_23.02.26»), а не внутренний ID.

## Создание МП (Монтажное Предложение)

### Из списка МП

1. Откройте **Коммерческие предложения → МП**
2. Нажмите **«Создать МП»**
3. На вкладке «Основные поля»:
   - Выберите **объект**
   - Выберите **контрагента** (исполнителя)
   - При необходимости укажите **связанное ТКП**

### Выбор монтажных смет

1. Перейдите на вкладку **«Монтажные сметы»**
2. Выберите одну или несколько монтажных смет для данного объекта
3. Суммы и человеко-часы из смет будут учтены в МП

### Условия для МП

1. Перейдите на вкладку **«Условия для МП»**
2. Условия «по умолчанию» уже выбраны
3. Вы можете убрать ненужные или добавить дополнительные

### Из ТКП

Вы можете создать МП напрямую из карточки ТКП:
1. Откройте детальную страницу ТКП
2. Нажмите **«Создать МП из ТКП»**
3. Выберите **Исполнителя** — отображаются контрагенты типа «Исполнитель/Поставщик» с подтипом «Монтажник» или «Оба»
4. Выберите **Монтажные сметы** — отображаются только монтажные сметы объекта текущего ТКП
5. Укажите сумму и трудозатраты (опционально)
6. Условия для МП подбираются автоматически (по умолчанию)
7. Система автоматически заполнит объект и привязку к ТКП

## Редактирование МП

Редактирование МП осуществляется прямо на странице детального просмотра (inline-редактирование):

1. Откройте детальную страницу МП
2. Нажмите кнопку **«Редактировать»** в правом верхнем углу
3. Все поля переключатся в режим редактирования:
   - **Название МП** — текстовое поле в заголовке
   - **Контрагент (Исполнитель)** — выпадающий список
   - **Дата МП** — поле ввода даты
   - **Статус** — выпадающий список
   - **Примечания** — многострочное текстовое поле
   - **Общая сумма** и **Трудозатраты** — числовые поля
4. После внесения изменений нажмите **«Сохранить»**
5. Для отмены без сохранения — нажмите **«Отмена»**

## Счета поставщиков в процессе расчёта

При подготовке сметы сметчик запрашивает счета и коммерческие предложения у поставщиков для расчёта стоимости материалов. Загрузка и проверка счетов выполняется прямо из редактора сметы.

> **Важно:** Счета, загруженные из сметы, являются «исследовательскими» — они привязаны к конкретной смете и **не видны снабженцу** в разделе «Счета на оплату». Это позволяет сметчику свободно загружать множество счетов для сравнения цен, не создавая хаоса в очереди оплат.

### Шаг 1: Загрузка счетов поставщиков

1. Откройте смету и перейдите на вкладку **«Счета поставщиков»**
2. Нажмите кнопку **«Загрузить счета»**
3. В появившемся диалоге выберите файлы одним из способов:
   - Нажмите **«Выбрать файлы»** и укажите нужные файлы
   - **Перетащите** файлы в область загрузки
4. Поддерживаемые форматы: **PDF**, **Excel** (XLSX, XLS), **изображения** (PNG, JPG)
5. Можно загрузить до **50 файлов** за одну сессию
6. Нажмите **«Начать обработку»**

Система автоматически:
- Распознает каждый файл через искусственный интеллект (LLM)
- Определит номер счёта, дату, поставщика, позиции и суммы
- Создаст карточку счёта в статусе **«Распознаётся»**, затем **«На проверке»**
- Если поставщик новый — автоматически создаст контрагента

### Шаг 2: Проверка распознанных счетов

После распознавания счета появятся в таблице на вкладке «Счета поставщиков» с индикацией статуса:
- **Фиолетовый** — «Распознаётся» (обработка ещё идёт)
- **Жёлтый** — «На проверке» (требуется проверка оператором)
- **Зелёный** — «Проверен» (данные подтверждены)

Для проверки:
1. Кликните по строке счёта в таблице — откроется страница детального просмотра
2. Слева отобразится PDF/скан документа, справа — форма для редактирования
3. Проверьте и при необходимости исправьте: контрагента, суммы, позиции
4. Нажмите **«Подтвердить»** — статус сменится на **«Проверен»**, товары из позиций будут добавлены в каталог
5. Нажмите **«Назад»** — вы вернётесь обратно на вкладку «Счета поставщиков» в смете

### Шаг 3: Подбор цен в смету

После того как один или несколько счетов проверены и подтверждены:

1. Перейдите на вкладку **«Строки сметы»**
2. Нажмите кнопку **«Подобрать цены из счетов»** (иконка волшебной палочки)
3. В диалоге нажмите **«Запустить подбор»**

Система проанализирует все проверенные счета и для каждой строки сметы покажет таблицу результатов:

| Колонка | Описание |
|---------|----------|
| ✓/✗ | Галочка — принять совпадение, крестик — отклонить |
| Строка сметы | Название позиции из сметы |
| Товар из каталога | Найденный товар-аналог из каталога |
| Цена | Цена товара из счёта поставщика |
| Источник цены | Поставщик, номер и дата счёта, откуда взята цена |
| Увер. | Уверенность совпадения (зелёный ≥80%, жёлтый ≥50%, красный <50%) |

4. По умолчанию приняты совпадения с уверенностью **≥80%**. Вы можете:
   - Кликнуть на галочку/крестик для ручного принятия или отклонения
   - Нажать **«Принять все >80%»** для сброса ручных изменений
5. Нажмите **«Применить (N)»** — выбранные цены будут подставлены в смету

### Сравнение цен от разных поставщиков

Для получения лучшей цены рекомендуется запросить счета у нескольких поставщиков:

1. Загрузите счета от 2–3 поставщиков через вкладку «Счета поставщиков»
2. Проверьте и подтвердите каждый счёт
3. При подборе цен система автоматически выберет последнюю цену для каждого товара
4. В колонке «Источник цены» вы увидите, от какого поставщика и из какого счёта взята цена

> **Примечание:** Счета, загруженные при подготовке сметы, являются справочными. На этапе закупки снабженец запросит собственные счета и самостоятельно отправит выбранный в реестр оплат. Исследовательские счета сметчика не мешают работе снабженца.

## Справочники

### Фронт работ и монтажные условия

Справочники «Фронт работ» и «Условия для МП» находятся в разделе **Справочники и Настройки → Фронт работ и монтажные условия**.

Каждый элемент справочника имеет:
- **Название** — описание пункта
- **Активен** — включён/выключен
- **По умолчанию** — автоматически выбирается при создании ТКП/МП

### Потенциальные заказчики

Новый тип контрагента **«Потенциальный Заказчик»** используется для хранения контактных данных клиентов, с которыми ещё не заключён договор.

Создать потенциального заказчика можно:
- В разделе **Справочники → Контрагенты** (выбрать тип «Потенциальный Заказчик»)
- При создании карточки в канбане поиска объектов

## Роли и права доступа

| Роль | Доступ |
|---|---|
| Менеджер по продажам | Канбан КП, создание ТКП/МП, работа со сметами |
| Маркетолог | Канбан поиска объектов, создание потенциальных заказчиков |
| Сметчик | Запрос счетов у поставщиков, подготовка ТКП/МП, работа со сметами |
| Снабженец | Прикрепление счетов поставщиков к карточкам канбана КП, отправка счетов в реестр |
| Руководитель | Полный доступ ко всем разделам |

## Часто задаваемые вопросы

### Как привязать несколько смет к одному ТКП?

На вкладке «Сметы» в форме создания ТКП вы можете отметить любое количество смет для выбранного объекта.

### Как работает «тоннель» между канбанами?

Это один борд с 12 колонками. Канбан маркетинга показывает 6 колонок, канбан КП — другие 6. Перемещая карточку между «границами» видимости, вы передаёте её между отделами.

### Можно ли вернуть карточку из канбана КП в канбан Маркетинга?

Да. Используйте выпадающий список перемещения — все 12 колонок доступны. Переместите карточку в любую колонку маркетинга.

### Видит ли снабженец счета, загруженные сметчиком из сметы?

Нет. Счета, загруженные через вкладку «Счета поставщиков» в смете, привязаны к конкретной смете и не отображаются в разделе «Финансы → Счета на оплату». Снабженец видит только обычные счета.

### Как подобрать цены для сметы из загруженных счетов?

Загрузите и проверьте счета поставщиков на вкладке «Счета поставщиков» в смете. Затем перейдите на вкладку «Строки сметы» и нажмите кнопку «Подобрать цены из счетов». Система найдёт совпадения по названиям и предложит цены с указанием источника.

### Что значит уверенность (%) при подборе цен?

Это оценка точности совпадения между строкой сметы и товаром из каталога. Зелёный (≥80%) — надёжное совпадение, жёлтый (50–79%) — требуется проверка, красный (<50%) — низкая уверенность.
